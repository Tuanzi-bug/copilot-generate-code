## 快速开始

### 创建新需求

使用 `/plan` 命令创建结构化需求文档：

```
/plan [描述你的功能或项目目标]
```

这将创建遵循专业软件工程实践的需求文档。

### 需求文档结构

```
requirement/
├── INDEX.md              # 本索引文件
├── in-progress/          # 进行中的需求
│   └── REQ-XXX-name.md
└── completed/            # 已完成的需求
    └── REQ-XXX-name.md
```

### 需求状态流转

```
新建 → 进行中 → 已完成
  ↓       ↓        ↓
in-progress/ → completed/
```

## 集成

### 与知识库集成

- **规划阶段**: 使用 `/plan` 创建需求
- **学习阶段**: 使用 `/learn` 记录实施经验
- **审查阶段**: 链接代码审查到需求文档

### 与 GitHub Issues 集成

- 需求可关联 GitHub Issues 进行跟踪
- 使用 `github-issues` skill 创建和管理 issues
- 在需求文档中引用 issue 编号

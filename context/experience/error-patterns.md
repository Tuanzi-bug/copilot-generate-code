# 错误模式

**Last Updated**: {日期}

## 概述

记录项目开发过程中遇到的常见错误及其解决方案。每个错误模式应包含完整的上下文、根因分析和可复用的解决方案。

> 🐛 **模板说明**: 使用 `/learn` 命令添加错误模式。每个模式应该具有可重复性和教育价值，而不仅仅是一次性问题的记录。

## 模板格式

```markdown
### {错误名称} - {简短描述}

**问题表现**: {用户看到的错误信息或现象}
**根本原因**: {为什么会发生这个错误}
**触发条件**: {什么情况下会出现}

**解决方案**:
```{语言}
// 修复代码示例
{代码}
```

**预防措施**: {如何避免再次发生}
**相关链接**: [文档](链接) | [Issue](链接)
```
